## Мета
Набуття навичок розробки власних контейнерів та використання ітераторів (Pattern Iterator).

## 1 ВИМОГИ
### 1.1 Розробник
- **Автор:** Ветріченко Дарина
- **Група:** КН-924з

### 1.2 Завдання
Розробити клас-контейнер `StringContainer`, що базується на масиві рядків.
Реалізувати:
- Динамічне додавання елементів (`add`).
- Видалення елементів (`remove`, `clear`).
- Пошук (`contains`).
- Інтерфейс `Iterable<String>` та власний `Iterator` (методи `next`, `hasNext`, `remove`).
- Заборонено використовувати стандартні колекції Java (`ArrayList` тощо).

---

## 2 ОПИС ПРОГРАМИ

### 2.1 Клас StringContainer
Це динамічний контейнер. Внутрішня реалізація базується на масиві `String[] data`.
Коли масив заповнюється, створюється новий масив більшого розміру (метод `resize`), і дані копіюються туди.

### 2.2 Реалізація Ітератора
У класі реалізовано внутрішній приватний клас `ContainerIterator`, який імплементує інтерфейс `java.util.Iterator`.
Це дозволяє використовувати контейнер у циклі `for-each`:
```java
for (String s : container) { ... }
public void remove() {
    if (lastRet < 0) throw new IllegalStateException();
    StringContainer.this.removeAtIndex(lastRet); // Виклик методу зовнішнього класу
    cursor = lastRet; // Коригування курсору
    lastRet = -1;
}
```
## ВИСНОВКИ
У ході виконання роботи було створено власний клас-контейнер StringContainer без використання Java Collections Framework.

Реалізовано механізм динамічного розширення масиву.

Опановано патерн Iterator: створено внутрішній клас, що дозволяє проходити по елементах контейнера та безпечно видаляти їх під час ітерації.

Закріплено роботу з масивами та системним копіюванням System.arraycopy.
