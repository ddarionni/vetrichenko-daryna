## Мета
Розробка модульних тестів з використанням бібліотеки JUnit 5. Перевірка коректності роботи контейнера та бізнес-логіки.

## 1 ВИМОГИ
1. Створити окрему директорію `test` для тестів.
2. Написати тести для класу `LinkedContainer` (методи `add`, `remove`, `get`, `size`, `iterator`).
3. Досягти покриття коду (Code Coverage) не менше 80%.
4. Написати тести для валідації (Regex) та логіки пошуку.

---

## 2 ОПИС ТЕСТІВ

### 2.1 Тестування контейнера (ContainerTest)
Використано анотацію `@BeforeEach` для ініціалізації нового контейнера перед кожним тестом.
Перевірено:
- Додавання та отримання розміру.
- Доступ за індексом (`get`) та обробка виключень (`assertThrows`).
- Видалення елементів з середини списку.
- Робота ітератора (цикл `foreach`).

### 2.2 Тестування логіки (ValidationTest)
Перевірено клас `RegexValidator`:
- Тести на коректні/некоректні імена (перевірка великої літери).
- Тести на валідацію дат (формат та діапазон).
- Тести на валідацію телефонів.

Також протестовано логіку підрахунку операторів (з Лаб 13) в ізольованому середовищі без використання багатопоточності.
