## 2 ОПИС ПРОГРАМИ

### 2.1 Засоби ООП та Параметризація
У роботі розроблено клас `LinkedContainer<T>`, який є узагальненим (Generic). Це дозволяє використовувати його для зберігання об'єктів будь-якого типу, а не тільки `AddressEntry`.

Структура даних базується на **двозв'язному списку** (або однозв'язному, залежно від реалізації), де кожен елемент зберігається у внутрішньому класі `Node<E>`.

### 2.2 Ієрархія та структура класів
Програма складається з таких класів:
1.  **`LinkedContainer<T>`** — основний контейнер.
    -   Внутрішній статичний клас `Node<E>` (містить дані та посилання на `next`).
    -   Реалізує інтерфейс `Iterable<T>`.
    -   Реалізує інтерфейс `Serializable`.
2.  **`Main`** — клас-драйвер для демонстрації функціоналу.
3.  **`AddressEntry`** — клас прикладної області (Контакт), що зберігається в контейнері.

### 2.3 Важливі фрагменти програми

**Внутрішня структура вузла (Node):**
```java
private static class Node<E> implements Serializable {
    E element;
    Node<E> next;

    Node(E element, Node<E> next) {
        this.element = element;
        this.next = next;
    }
}

```
