## Мета
Ознайомлення з моделлю потоків Java. Організація паралельного виконання трьох незалежних задач з обробки даних.

## 1 ВИМОГИ
1. Створити мінімум 3 паралельні потоки для обробки колекції:
    - Підрахунок абонентів Vodafone.
    - Підрахунок мешканців Києва.
    - Пошук найстаршої людини.
2. Алгоритми не повинні змінювати колекцію (read-only).
3. Реалізувати механізм **таймауту**: якщо обробка триває довше заданого часу, потоки примусово зупиняються.

---

## 2 ОПИС ПРОГРАМИ

### 2.1 Клас Thread та Runnable
Для кожної задачі створено об'єкт `Runnable` (через лямбда-вираз).
Потоки запускаються методом `start()`.

### 2.2 Імітація навантаження
Оскільки сучасні процесори обробляють тисячі записів миттєво, у цикл обробки додано `Thread.sleep(1)`, що створює штучну затримку в 1 мс на кожному записі. Це дозволяє наочно продемонструвати роботу таймауту на вибірці у 5000 записів (що займе приблизно 5 секунд).

### 2.3 Зупинка потоків (Interruption)
Використовується безпечний механізм зупинки.
Головний потік чекає завершення роботи через `t.join(timeout)`. Якщо час вичерпано, він викликає `t.interrupt()`.
Всередині робочих потоків вставлено перевірку:
```java
if (Thread.currentThread().isInterrupted()) {
    return; // Негайний вихід з методу
}
```
